Python缩略语和词汇

原文：https://medium.com/analytics-vidhya/python-acronyms-and-vocabulary-bd2cd0c53bb6?source=collection_archive---------20-----------------------#2020-04-03

这是一个简短的词汇表，当你开始编码时会弹出来，但可能不是100%清楚。词汇是可能有助于让Python对初学者来说看起来很神秘的东西的一部分(其他的在这里)。在这里，我只关注首字母缩略词。好吧，你可以谷歌一下，但我会给你留着的。我还会或多或少地把它们分成不同的区域。

Python(通用)

  • PEP 。Python增强提案”。Python就是这样进化的。您提出了一个改进，如果获得批准，它将在下一个版本中出现。(实际上，事情要复杂得多，参见 PEP 13 )。pep是一种你需要习惯学习的文档，把它们想象成特殊的文档(docs)。
  • BDFL 。仁慈的终身独裁者。用一种友好的方式来称呼Python的创造者吉多·范·罗苏姆
  • IDE 。“集成开发环境”。一个帮助你完成(编码)工作的程序。对于R think RStudio 。在Python中认为 PyCharm ， IDLE ， Spyder 。最后两个实际上是首字母缩写词。
  • 无所事事。“集成开发和学习环境”的递归首字母缩写，其中一个成员叫做埃里克·艾多尔。这是标准Python发行版附带的IDE。
  • Spyder 。anaconda IDE本身就是一个缩写:“科学PYthon开发环境”。它会给你很多科学的Python库。
  • Venv 。虚拟环境。隔离工具。它是你构建你的程序来避免任何改变，如果外部环境破坏了你的代码，而你不知道如何和为什么。

界面、文件扩展名和网络相关资源是大写字母的另一个来源

  • GUI 。图形用户界面。使用图形、指向和点击与程序交互。想想:猴岛。
  • CLI 。命令行界面。现在，我们只使用通过命令行界面传递的文本命令与机器进行交互。想想:黑客帝国的第一幕。您可以将CLI称为Bash、Shell或Terminal。
  • CMS 。“内容管理系统”。一个帮助你传递内容的系统。想想Blogger或Wordpress。
  • CSV 。逗号分隔值。存储数据并在软件间传递数据的避风港。
  • JSON 。JavaScript对象表示法。交换数据的标准，既可读又对机器友好。
  • REPL 。“读取-评估-打印循环”。也称为“语言外壳”。这是一个交互式计算机编程环境，它接受输入(“read”)，执行输入(“eval”)并发送输出(“print”)。永远都是(循环)。
  • 宋承宪。“安全外壳”。一种加密网络协议，用于保护不安全网络上的网络服务。思考:避免日志或类似的东西，如果没有的话。
  • 网址。“统一资源定位器”。正常人的“网址”的具体名称。
  • 休息。代表性状态转移。这是罗伊·菲尔丁在他的博士论文中定义的。RESTful服务以特定的方式使用HTTP动词(即GET、HEAD、PUT)。REST用于提供API(参见)。
  • API 。应用程序接口。软件的计算接口。你调用一个API与系统交互。(例如:所有的Twitter API都有大数据流，你不能独自在搜索栏中工作。)
  • YAML 。递归缩写:“YAML不是标记语言”。YAML实际上是一种标记语言，旨在让人类和机器都感到愉快。

开发原则、技术和哲学也导致了相当多的缩写:

  • YAGNI 。你不会需要它的。极限编程的原则，根据这个原则，你只需要在真正需要的时候添加东西和函数。
  • XP 。极限编程。它与Windows以前的版本没有关系。这是敏捷软件开发的一种实现，根据只有当(并且如果)你真正需要的时候，你才实现新的东西。
  • TDD 。测试驱动开发。你不写代码。您首先编写指导代码开发的测试。如果你的代码通过了测试，你就可以继续开发。否则你会试着失败，你会考得更好…你可能听说过测试羊，是吗？
  • BDD 。“行为驱动开发”。用一个不那么吓人的词重新推销TDD。黄瓜书《面向测试人员和开发人员的行为驱动开发》使其广受欢迎。
  • 固。这是一整套与面向对象编程的坚实原则相关的缩略语。大致有:单责任、开-闭、利斯科夫替代、接口隔离、依赖倒置。很可怕，是吧？
  • VCS 。版本控制系统。这基本上意味着像Git和GitHub这样的工具。但是一定要看看Maven。

不是首字母缩写，而是:

  • 单据。文档——如"阅读文档"文档是你找到关于软件包的技术信息的地方。
