组织和就业市场不断地将“数据科学”和“人工智能”贴在与数据相关的一切事物上，这往往会导致对成为一名有效的数据科学家需要哪些技能的困惑。让我们通过添加“全栈”一词来使它变得更加模糊，并尝试定义它。在这样做的同时，让我们来看看这个行业在人工智能项目中面临的一个普遍问题。

**“大多数商业分析和人工智能项目仍未能投入生产或产生预期影响”**

让我们探讨一下造成这种情况的一些主要原因:

1.  数据稀缺
2.  糟糕的数据治理
3.  缺乏基础设施和人才
4.  **协作**

是的，人工智能项目因为缺乏协作而失败，这与你通常在人工智能项目团队中发现的人的本性有关。

1.  **业务领导/项目经理**

![](img/b67bb43761307e79f3cb6a913a59ec83.png)

这些人有很好的沟通技巧，从商业角度对需要实现的目标有全面的了解。但是，他们很少了解人工智能/软件的能力，主要是其局限性。这导致团队/公司过度销售他们的能力，并最终意识到开发他们签约的东西需要大量的 R&D 工作和基础设施。

**2。数据科学家**

"*我需要 32g 的特斯拉 P100 来训练 100 张唱片的伯特模型。*

能够建立复杂的机器学习和深度学习模型的数学/统计专家。他们中的许多人都有一种“闪亮的东西疾病”，并且痴迷于尖端技术和艺术模型。由于对业务问题和部署选项一无所知，他们有时会以“用大锤砸死一只蚂蚁”而告终，并错失大局。

**3。软件/部署工程师**

这些是您的软件/开发工程师，他们在生产中部署您的 ML/DL 模型。有时，他们会忘记业务问题和数据科学管道，最终很难将模型与应用程序的其余部分集成在一起。

为了弥合这一差距，该行业需要了解端到端人工智能生命周期的人，这就是全栈数据科学家介入的原因。所以，让我们继续给这个名字下一个定义。

# **谁是全栈数据科学家？**

与软件开发中的前端后端“全栈”相反，全栈数据科学家可以将数据驱动的概念从识别/构思到执行，从而带来一些有形的、可测量的和有影响的改进。能够从头到尾推动所有技术和业务方面的人。

*免责声明:全栈数据科学家也被称为“独角兽”，这是有原因的，“他们不存在”。随着不断变化的技术堆栈、新工具/框架以及大型技术公司使用 GPU 堆栈来创建新的艺术架构，不可能成为“万能大师”。话虽如此，但可以实现的是开发一套 T 型技能，正如 Ankit Rathi 在他的* [*博客*](https://ankit-rathi.github.io/blog/markdown/2020/03/26/5-tips-for-ds-ai-beginners-enthusiasts.html) *中所指出的。你专攻一项技能，对其他技能的了解足以让你变得危险。*

所以让我们切入正题，谈谈成为一个人所需要的技能和一些资源。

## 6.模型部署

一个准确率 99%的模型，如果你不能部署它，不能产生商业价值，那就是徒劳的。部署是在您开始构建模型之前就应该充分考虑的一个部分，忽视它的项目注定会失败。需要回答的一些问题:

*   我的部署批次是实时的，还是在边缘设备上？
*   我的延迟要求是什么？
*   我是否应该转向不同的语言，如 C++，Java？
*   我需要多大的计算能力？我需要一个 GPU 吗？
*   我如何扩展我的管道？
*   云还是原生？如果是云，是哪个提供商和服务？

在生产中高效部署模型时，熟悉 Docker、Kubernetes 等容器服务和云提供商的部署服务变得至关重要。

## 8.测试、实验跟踪和生产后监控

*   **测试**:在传统的软件系统中，人类编写与数据交互的逻辑来产生期望的行为。我们的软件测试有助于确保这种书面逻辑与实际的预期行为一致。然而，在机器学习系统中，人类在训练期间提供期望的行为作为示例，并且模型优化过程产生系统的逻辑。确保这种学习到的逻辑能够始终如一地产生我们想要的行为是测试 ML 系统变得棘手的地方。
*   **实验跟踪**:人工智能项目是高度迭代的，因此跟踪你的实验(模型架构、超参数、性能度量)是必要的。以有组织的方式跟踪实验有助于解决此类项目的所有核心问题。权重和偏差(wandb)和 MLFlow 是一些帮助个人跟踪其实验的简单工具。
*   **模型监控**:这是机器学习生命周期中的一个操作阶段，在模型部署之后，它需要“监控”您的 ML 模型，例如错误、崩溃和延迟，但最重要的是，要确保您的模型保持预定的预期性能水平。了解用于监控的流行指标将非常有用。

## 9.软件工程基础和最佳实践

与过分强调数据科学课程和社交媒体相反，数据科学家花费不到 10%的时间来构建和调整模型。剩下的时间花在软件工程任务上，如数据收集、数据处理、部署和维护代码库。所以你有必要了解以下内容。

*   版本控制(代码和数据)
*   应用程序接口
*   使用命令行

## 10.商业头脑和有效的沟通

现在，这是一项来自经验、广泛阅读和实践的重要技能。对于每个企业来说，让产品或服务变得更好是数据科学项目的最终目标。不考虑这一点是荒谬的。您的数据团队可能拥有最好的编码人员和最好的统计人员，但是如果他们不知道他们的数据项目的实际业务应用，整个事情将毫无意义。还有相似性，你的分析再伟大也没用，除非你能解释给别人听。
