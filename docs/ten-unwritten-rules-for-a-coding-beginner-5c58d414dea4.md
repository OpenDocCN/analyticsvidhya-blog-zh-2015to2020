## 一个“易于遵循”的新手程序员规则手册。

![](img/2711b17efa18f51517433f91256d2bc6.png)

卢卡·布拉沃在 [Unsplash](https://unsplash.com/s/photos/coding?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上的照片

进入本世纪的第三个十年，编程是最需要的技能之一。随着越来越多的数字趋势的流入，编码在技能市场上变得越来越突出。因此，每年都有许多新的编码者进入市场…我们从自己的经验中学习，但我们也可以从别人的经验中学习。

所以，如果你是新手，试着遵循这些对我非常有效和有帮助的规则。希望他们也能帮助你。

# **1。不必匆忙，但没有理由放慢脚步**

固定每天的小时数，并全身心地投入其中。别担心，你才刚刚开始。会顺其自然的。但是千万不要跳绳，哪怕一天跳绳都要三思**。这很容易成为一种习惯。**

# **2.学会实施**

**永远不要读完一个话题，然后继续下一个话题。学习它来实现它。**看一个题目，想一个可以用的情景，应用。**如果想不出什么，谷歌一下，写点短代码，调试运行。但是永远不要离开这个话题，直到实现它，否则你很可能会忘记它。**

# ****3。基础是基础****

***“地基不牢，建不成摩天大楼”。*基础薄弱就走不到高。**读基础，应用基础，复习基础**。你必须对库、OOPS 概念、数据类型、内存分配以及这门语言的所有其他基础知识有很好的了解。**

# **4.始终将代码放在类和函数中**

**这很重要。无所谓代码是**两行**还是**两百行，**通过使**函数**至少**如果不是类**来编写。不要写很长的代码段落。保持结构化，而不是自由式。***是码，不是嘻哈”。*****

# **5.不要复制粘贴代码**

**记住，**你是新手，不是专业人士。**自己写，千万不要复制粘贴。复制-粘贴使这项工作变得快速而简单，但是作为一个新手，你会不断忘记它，当你写大代码，真正的代码时，它会减慢你的理解速度。**

# ****6。错误是好的，错误是重要的****

**代码中出现错误？别担心。没有人能编写没有错误的代码。拥抱错误，从错误中学习。耐心点，调试错误，思考，如果得不到答案，就在 Stackoverflow 上搜索。但是尽量不要多次重复同样的错误。**

# **7.不要只是学习，而是生活**

***“人类是群居动物”。*无论你有多内向，你都会在社会化中生存。编程也是如此。**加入脸书群**、**看编码博客、看教程、看评论**、**那里**、**关注 Instagram、Discord、Reddit 上的编码迷因页面。**有一个完整的编码社区，成为它的一部分吧。**

# **8.免费项目是金矿**

**互联网上有成千上万的免费项目，有一个完整的社区为你工作，为像我们这样的人工作。**阅读免费项目，查看代码流程。看看专业人士是怎么做的。探索 Github，尝试理解它。不要紧，如果你是新的，你会得到它在一段时间内给一个尝试。****

# **9.将你的代码分成更小的部分，在重要的行上添加注释**

***试着写一段复杂的代码，一个月后读它，你将无法理解你自己的代码***

**永远用小代码写，小代码快速，易管理，干净，容易理解。添加注释来定义您正在做的事情。这使得**代码维护更加容易**。记住，你不是唯一一个阅读和处理你的代码的人。**

# **10.像和尚一样坐着，不要像猴子一样**

**是的，这很有趣，但它有太多的价值。问问那些长时间坐着的人。**姿势很重要**。只有身体健康才能工作。长时间编码**坐姿不当**会造成严重的**姿势畸形**以及**心血管疾病**。最突出的是**心皮隧道。**间隔做**拉伸运动，保持水分，按时吃饭**保持健康和身材。这是一项久坐不动的工作，但对脑力的消耗太大。定期放松，每天锻炼。**

## ****最后但同样重要的是:****

# **为编码社区做贡献:**

**你从他们那里拿走了太多，如果没有社区的支持，程序员就不能成为程序员。这是一项无私的工作，一项诚实的工作。就当是回报吧。回答各种平台的查询，给 Github 投稿，在编码上帮助人。**
