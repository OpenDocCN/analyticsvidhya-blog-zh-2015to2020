数据分析的5v

原文：https://medium.com/analytics-vidhya/the-5vs-of-data-analysis-2cb0c83a91cc?source=collection_archive---------23-----------------------#2020-03-15

数据分析是收集、编译、转换、处理和分析数据的过程，以便您可以使用它来获得更深入的见解并做出决策。如今，组织使用这些见解来做出决策，如何时推出产品、在哪里营销以及许多其他用例。基于组织为其用例收集和处理的数据类型，这也带来了许多挑战。

数据分析的5个方面不仅定义了有效系统的构建模块，还可以帮助组织了解他们在从数据中获得洞察力的数据管道中存在的问题。让我们首先在设计系统的背景下定义 5 V。它们是:

  1. 容量:系统能够处理的数据量。
  2. 速度:系统能够处理数据的速度，即数据流入和流出系统的速度。
  3. 多样性:由于企业从多个来源收集数据，并且所有来源可以提供不同格式的数据，因此多样性可以定义为系统处理各种类型数据的能力。
  4. 准确性:由于数据有可能来自不同的系统和不同的阶段，准确性保证了数据的完整性，没有在任何先前阶段引入的偏差。
  5. Value :确保系统能够从数据中传递出值。

现在我们知道了系统环境中的 5 V是什么，那么让我们看看如何利用它们来理解组织在当今以数据为导向的世界中所面临的问题:

  1. 数量:当一个组织拥有的数据超过它能够处理的数量时，即处理和分析，他们就有数量问题。
  2. 速度:当现有的系统不能处理组织所需的洞察力或新特性的速度时，他们就遇到了速度问题。
  3. 多样性:当一个组织的管道中有许多类型的数据源，但是他们没有办法处理所有的类型，那么他们就有一个多样性的问题。
  4. 准确性:由于数据来自许多阶段，并且可以转换成许多不同的形式，使其对组织中不同类型的角色和用户有用，因此保持数据的真实性并且之前的任何阶段都没有在数据中引入任何偏差非常重要。如果在这个过程中的任何一个阶段产生了偏差，那么组织就有一个准确性的问题。
  5. 价值:你有没有听说过组织说我们拥有海量数据来支持我们的小功能列表？嗯，这些组织很有可能存在价值问题，因为他们拥有大量数据，却忽略了数据的价值。
