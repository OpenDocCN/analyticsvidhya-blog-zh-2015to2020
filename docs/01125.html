它是怎么做到的？

目前，小行星能够从二维数据中地形化一颗具有可选集群属性的行星。它仍然是非常新的，并将在功能上不断增长，但现在当terraforming一个新的世界时，图书馆可以实现以下目标:

  • 在空间的某个地方生成来渲染你的作品
  • 基于输入数据统计生成生态(您也可以手动指定生态)
  • 生成具有地形细节(等高线)和地貌细节(坡度)的地块
  • 以山体阴影的形式生成照明效果

作为一个用户，你可以控制你生成的小行星的细节层次，并且可以选择在交互式绘图中渲染什么。完整的公共API记录在这里。

在引擎盖下，我正在使用 plotly 渲染小行星。目前，我使用的ScatterGeo trace没有z感知功能，只能处理矢量数据，所以我必须想出有趣的方法来给人以3D深度的错觉，就像打印地图一样。

安装就像pip install planetoids一样简单，而且 项目 已经用Travis-CI建立了自动化测试，用Codacy建立了自动化代码审查&分析，使对这个项目感兴趣的其他人更容易加入并做出贡献。我已经标记了一些 好的第一期 给任何有兴趣帮助我扩展项目的人。😁

它仍然处于一个 alpha 状态，所以随着特性的增加和完善，API的某些方面可能会在未来的版本中发生变化。

网上有少数互动 演示 笔记本 可以帮助你入门。我非常想知道人们最终会用这个库产生什么，我鼓励任何人提交演示来作为库的一部分，以展示如何在不同的领域中使用它。
