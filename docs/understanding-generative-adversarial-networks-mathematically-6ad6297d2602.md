# 摘要

在机器学习世界中，生成性对抗网络已经成为一件大事。在这篇论文中，我们来看看生成性对抗网络如何工作背后的全部数学来源。我们从最基础的开始，给出一个直观的理解并定义一个生成性对抗网络的不同部分，一直到从数学和概念上理解损失函数。

# 定义生成器

生成式对抗网络由两个不同的部分组成:鉴别器和生成器。通过函数给定生成器: **G(z) = x** 。发生器函数通过 **G(z)** 表示，其中 **z** ，发生器的参数，是添加到数据集的噪声。输出变量 **x** 代表应用噪声后新生成的数据。

# 定义鉴别器

鉴别者是生成性对抗网络的第二部分。鉴别器可以通过函数来表示: **D((G(z)))** 。鉴别器函数在数据集上被训练后，基于其先验知识输出一个值。鉴别器输出一个标量值，例如: **0 < D((G(z))) < 1。**

如果 **D(G(z))** 产生一个更接近于 1 的值，那么生成器已经完成了它的工作，并且产生了一个看起来不是合成的图像。如果 **D(G(z))** 更接近 0，则鉴别器将图像识别为合成数据。重复该过程，直到鉴别器不再能够区分合成图像和真实图像，因此输出更接近 1 的数字。

# 最小最大损失函数

传统的生成对抗网络由两个损失函数组成。一个损失函数用于鉴别器，一个用于发生器。结合起来，他们创造了极大极小损失函数。最小最大损失函数定义如下:**E _ x[log(D(x))]+E _ z[log(1—D(G(z)))]**。

在这个损失函数中，有几个部分。 **D(x)** 代表实际数据的鉴频器值。代表所有真实数据的期望值。 **D(G(z))** 是给定噪声 **z** 进入发生器时，产生数据的鉴别器值。 **E_z** 代表所有生成数据的预期值。 **E_x[log (D(x))]** 用于更好地识别真实图像，而 **E_z[\log (1 — D(G(z)))]** 用于更好地识别生成的图像。

# 最小最大损失函数目标

GAN 发生器的最终目标是最小化 Minimax 损失函数，而鉴别器试图最大化损失函数。这也可以直观地理解，因为随着 **D(G(z))** 接近 1，鉴别器认为生成的图像是真实的。同时， **E_z[log (1 — D(G(z)))]** 趋近负无穷大。另一方面，如果 **D(G(z))** 接近 0，鉴别器识别出生成的图像是假的。这将导致**E _ z[log(1-D(G(z)))]**也达到接近 0，这意味着最小最大损失函数被最小化。

# 瓦瑟斯坦损失函数

Wasserstein 损失函数是另一种损失函数，通常用于生成式对抗网络。使用 Wasserstein 损失函数的 GAN 称为 WGAN。瓦瑟斯坦损失函数有两个不同的部分。鉴别器损失函数，也称为 critic，定义如下: **D(x) — D(G(z))** 。发电机损耗函数定义如下: **D(G(z))** 。

# 理解瓦瑟斯坦损失

分开来看，最容易理解瓦瑟斯坦损失函数。仅作为评价函数，鉴别器旨在通过最小化 **D(G(z))** 来最大化评价值。另一方面，生成器的目标是最大化 **D(G(z))** ，因此造成了这种对抗性。

通过理解生成对抗网络背后的数学推理，可以对它们的功能有更直观的理解。在整篇论文中，我们不仅着眼于生成性对抗网络的最大简化，而且还努力从数学和概念上理解损失函数的基础。通过这种方法，对生成性对抗网络的后端有了更充分的了解。
