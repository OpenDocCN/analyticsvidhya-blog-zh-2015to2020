为什么整理数据？

原始数据很难操作、建模和可视化。这是因为原始数据是杂乱的。大量时间和精力花费在准备数据上，以便为操作、建模和可视化做好准备。我将谈论处理混乱的数据，即整理混乱的数据。整理数据的想法在Hadley Wickham的整理数据论文中有很好的阐述。

这是一个关于整洁数据主题的介绍。如果你熟悉它，你可以自由跳转到后续文章，进入整理凌乱数据的细节。

什么是整齐的数据？

如果数据满足以下条件，则称其为整洁的:

  1. 每个变量形成一列。
  2. 每个观察形成一行。
  3. 每种类型的观测单元形成一张表。

因此，如果行、列和表与观察值、变量和类型的匹配方式不符合上述任何条件，那么数据就是杂乱的。用一个例子可以更好地说明这一点。

不整洁的数据示例

我将用一个假设的数据来衡量五个学生在三个学期的两次考试中的表现。

由于以下常见的不整洁数据问题，这些数据是不整洁的:

  • 有些列标题是值，不是变量名。

列标题期限1 、期限2 和期限3 为数值。这些不必是列标题，而是另一列中的值，如术语。

  • 多个变量存储在一列

性别和年龄一栏有两个变量，显然是性别和年龄。例如 Mike在该列的值是 m_12 ，这意味着他是一名12岁的男性。性别和年龄值需要在不同的列中。

  • 变量存储在行和列中

测试号栏不是一个变量，它存储了变量测试1 和测试2 的名称。因此，test 1和test 2需要是列的标题，因为它们是变量。

  • 多种类型的观测单位存储在同一个表中

上表有两种观察单位的数据:每学期两次测试中的学生和学生表现。这些不同的观察单位需要在它们自己的表格中保持一致。因此，学生用表应该有 id 、姓名、电话、性别和年龄。性能表应该有剩余的数据，加上在两个表中通用的 id 列。

这标志着一个杂乱数据探索的结束。然而，这些并不是造成数据混乱的唯一问题。这里还有很多我没有涉及的问题。
