# 从 Java 到 JavaScript 还是 Java 和 JavaScript？

> 原文：<https://medium.com/analytics-vidhya/from-java-to-javascript-or-java-and-javascript-1bcdaf847bc9?source=collection_archive---------29----------------------->

Java 和 JavaScript 不再是青少年了，它们是成熟的语言，除了几乎是双胞胎之外，因为它们拥有相似的名字，这引起了很多困惑，直到今天，它们几乎是在同一年诞生的，至少对于专家来说基本上已经消亡了，因此在这些年里复兴了很多，今天它们毫无争议地成为与 Phyton 一起登上领奖台的三种最重要的语言之一，但它们是非常不同的语言

当我开始编程时，javascript 仅仅是用来在前端执行动态的事情，是的，那是很久以前的事了。但是多年来我一直听说 javascript 在服务器端运行，我必须承认，这很难相信，特别是如果你和我生活在同一个时期，我们不认为 javascript 是一门严肃的语言或者根本不是一门语言。

有一天，不知从哪里，我开始思考一些语言是如何失去空间的，以及他们的程序员是如何没有意识到这一点的。今天没有任何迹象表明这种事情会发生在 Java 上，但是我们永远不知道，对吗？！。带着这个想法，我决定尝试动量语言，这里有很多人说我们可以做任何事情。

第二次忏悔的时间到了，是的，他们是对的，JavaScript 很棒。

正如我所说的，我已经使用 java 很长时间了，超过 14 年，但是在使用 JavaScript 几天后，我开始问自己“既然 JavaScript 编程更快，我们可以在云上节省大量资源，为什么要使用 Java 而不是 JavaScript？”。

## NODEJS

NodeJS 创建于 2009 年，现在被数千家公司使用，如 T2 网飞、Paypal 和 LinkedIn T3，如果你花几个小时阅读和使用它，很容易理解为什么 NodeJS 会发展这么快，启动和使用都很简单。澄清一下，我并不是说你会在几周内成为一名高级绝地 javascript 开发大师，但是你将能够在比其他语言(例如 Java)更短的时间内开发出优秀的东西，但是一如既往，这只是我的观点，基于我迄今为止的个人经验。

学习 JavaScript 和 NodeJS 还帮助我更好地理解了 Java 所遭受的变化，并认识到了类似 Java 8 中发布的函数表达式和 Java 10 中出现的“var”单词概念。

是的，我知道，我只是说了 JavaScript 的优点。您一定想知道，JavaScript 是如此受追捧的银弹吗？到目前为止，我还没有发现 JavaScript 不能很好解决的问题，但是专家们总是说这种事情不存在，所以让我们继续和他们在一起，很有可能他们发现或者需要一些我不知道的特定的事情或者情况。

我已经说了很多关于 JavaScripts 的好东西，但是没有关于 Java 的，但是不要误解我。我喜欢 Java，毫无疑问，Java 是一门非常非常好的语言，也许我只是对一个孩子有了新玩具时的新事物感到兴奋。我继续在日常工作中使用 Java，对于那些想学习 Java 的人，我强烈推荐，特别是因为现在我们有了很好的选择来开发 Java 世界，比如 Spring 和 Quarkus，在那里你可以在几分钟内创建一个 rest API。

写这篇文章的原因是，我不清楚一个比另一个更好的地方/时间，或者我/我们是继续使用 Java 还是跳到 JavaScript。也许问题是我们两者都需要？还是他们解决了同样的问题？
