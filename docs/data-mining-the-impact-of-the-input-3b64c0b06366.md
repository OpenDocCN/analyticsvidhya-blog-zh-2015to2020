我认为对于任何软件系统来说，理解输入和输出是什么远比知道它们之间发生了什么更重要，数据挖掘也不例外。

输入采用*概念、实例、*和*属性的形式。*

因此，在这篇文章中，我解释了这些术语，并谈到了准备数据。

# **什么是概念？**

![](img/c34e4ba266a50f1878a2bae524fdf48b.png)

照片由 [NeONBRAND](https://unsplash.com/@neonbrand?utm_source=medium&utm_medium=referral) 在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

在数据挖掘应用中出现了四种基本不同的学习方式。

在*分类学习中，*学习方案以一组已分类的示例呈现，期望从中学习一种对未见过的示例进行分类的方法。

在*关联学习中，*寻找特征之间的任何关联，而不仅仅是预测特定*类*值的关联。

在*聚类中，寻找属于一起的*组实例。在*数值预测中，*要预测的结果不是离散类而是数值。

不考虑学习的类型，我们称要学习的东西为概念，由学习方案产生的输出为概念描述。

机器学习方案的输入是一组实例。

这些实例是要被分类、关联或聚集的事物。

尽管到目前为止我们称它们为*实例，但从今以后我们将使用更具体的术语*实例*来指代输入。*

每个实例都是要学习的概念的一个单独的、独立的例子。

# **准备输入**

为数据挖掘调查准备输入通常会消耗整个数据挖掘过程中投入的大部分精力。

虽然这篇文章实际上不是关于数据准备的问题，但是我想让您对所涉及的问题有一个感觉，这样您就可以理解其复杂性。

痛苦的经历表明，真实数据的质量往往令人失望地低，仔细检查，一个被称为*数据清理的过程，*多次都有回报。

## **收集数据**

整合来自不同来源的数据通常会带来许多挑战，不是深层次的原则问题，而是令人讨厌的实践现实。

不同的部门将使用不同的记录保存方式、不同的约定、不同的时间段、不同程度的数据聚合、不同的主键，并且会有不同种类的错误。

数据必须被组装、集成和清理。

公司范围的数据库集成的想法被称为*数据仓库。*

数据仓库提供了对公司或组织数据的单一一致的访问点，超越了部门划分。

它们是旧数据发布的地方，旧数据可以用来为业务决策提供信息。

## **缺失值**

实践中遇到的大多数数据集都包含缺失值。

缺失值通常由超出范围的条目指示，可能是通常只有正数的数值字段中的负数(例如-1)，或者是通常不可能为 0 的数值字段中的 0。

对于名义属性，缺少的值可以用空格或破折号来表示。

有时会区分不同类型的缺失值(例如，未知值、未记录值和无关值)，并可能用不同的负整数(-1、-2 等)来表示。).

## **不准确的数值**

数据变得陈旧。许多项目随着环境的变化而变化。

例如，邮件列表中的项目:姓名、地址、电话号码等等。经常换。

您需要考虑您正在挖掘的数据是否仍然是最新的。

# 结论:

数据清理是一个耗时耗力的过程，但对于成功的数据挖掘来说是绝对必要的。

对于大型数据集，人们通常会放弃。

他们怎么可能全部检查完呢？

相反，您应该抽取一些实例并仔细检查它们。

你会对你的发现感到惊讶。查看数据的时间总是值得的。
